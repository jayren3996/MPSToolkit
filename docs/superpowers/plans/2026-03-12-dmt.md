# Operator-Space DMT Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add operator-space DMT to `MPSToolkit` with faithful 2-site and 3-site kernels, a generalized contiguous-window implementation, focused tests, and a comparison benchmark validating that the generic routine recovers the specialized cases.

**Architecture:** DMT lives under `src/operator_space/dmt.jl` as an operator-space truncation backend layered on top of existing Pauli-basis helpers and TEBD-style local gate application. The first implementation keeps specialized 2-site and 3-site entrypoints while factoring shared environment and constrained-truncation helpers into a generic windowed routine that can be benchmarked directly against them.

**Tech Stack:** Julia, ITensors, ITensorMPS, Test
